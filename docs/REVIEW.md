# Agent Brain — Enhancement Report (synthesis of 5 lenses)

## 1. Verdict

**Sound in shape, not yet sound to build.** The model (feature-as-identity, pull-not-push, M9/M10, filesystem-only) is right and the simplicity lens is correct that it's mostly right-sized. But four of five lenses independently verified two hard blockers that break the core loop *today*: (a) root discovery via `git rev-parse --show-toplevel` resolves to each *sibling* worktree's own (non-existent) `.brain/`, silently re-introducing the exact cross-worktree-visibility problem the dropped MCP server existed to solve; and (b) the `*<slug>*` wildcard identity already mis-resolves 3 of the 4 real features (cockpit double-matches, workbench has no slug, langgraph lives on `main`). Around those sit a cluster of mediums that must be reconciled before the mechanisms fit together (gitignored nav skill, no read cursor, undefined collision-match semantics, reconciliation pinned to a possibly-parked `/wrap`). All are fixable inside the filesystem-only / M9 / M10 / lightweight constraints — most by *subtraction* or one-line corrections. Fix the before-build set, honor the during-build notes, and this is buildable as v1 with the superset as a clean Phase-2 add.

## 2. Adopt — before build (change the spec first)

**A. Root discovery — `--git-common-dir`, not `--show-toplevel`.** `root = realpath "$(dirname "$(git rev-parse --git-common-dir)")"` (canonicalize — common-dir returned bare `.git` from main here). One `brain_root` function reused by `whoami`/`announce`/`status`/all hooks. *Verdict: ADOPT — one-line, strictly more portable, the spec doesn't work without it. Blocker.*

**B. Identity model — anchor the match, keep multi-pattern, add a non-branch fallback.** (i) Match a *delimited token* after the ticket (`feat/<ticket>-<slug>-…`), not a bare substring → kills `cockpit-mini-graph`→graph and `graph`⊂`langgraph` collisions. (ii) `whoami` on multi-match disambiguates by `current_worktree` (unique at any instant), else **refuses ambiguous** rather than silently picking. (iii) `owns_branches` stays a *list* and `brain new-feature` seeds it deterministically from the real branch — workbench gets `*board-composer*`, not a slug that isn't there. (iv) Main-resident features (langgraph) get a non-branch identity via a `BRAIN_FEATURE` marker set at SessionStart; document "slug-in-branch" as the *going-forward* convention with `main`-resident as the named exception. *Verdict: ADOPT — identity is broken for 75% of live features as written. Blocker.* **This is the one place I rule against the simplicity lens** (which called the multi-pattern/append path "dead guidance"): it is not dead — it is load-bearing for the messy present.

**C. Self-onboarding pre-check (closes the chicken-and-egg + the unreachable M6 guard).** Add one SessionStart step that runs *before* the whoami allowlist: on `whoami` empty **but** branch is `feat/…`-shaped and not a known non-brain worktree → inject "no presence note — run `brain new-feature <slug>`" (slug parsed from branch). This is the only thing that keeps M9 intact (no human runs `new-feature`) and is also where the M6 slug guard actually fires. *Verdict: ADOPT before build — without it every new feature is invisible forever. High/blocker.*

**D. Collision-match semantics — one rule.** Entry ending `/` = zone (prefix-match the changed path); otherwise exact-file. Apply uniformly to `touches` and connection `files`. The hook is unimplementable and the two motivating examples (`agent.ts` exact vs `_views/` prefix) can't both work without it. *Verdict: ADOPT before build — one sentence in the schema + nav skill. High.*

**E. Defer the PostToolUse "fast-awareness" tier; fix the journal cursor.** Ship **SessionStart + PreToolUse only** (both day-one wins still land). This simultaneously removes the surplus third hook (simplicity) *and* sidesteps its real bug — mtime+grep re-injects already-seen lines because there's no read cursor and `updated` is date-granular. Then: make `updated` a full ISO timestamp (cheap, also sharpens liveness), and **name tier-1's owner** — SessionStart/`brain status` must surface relevant recent journal lines (today it's scoped to agents+connections only). *Verdict: ADOPT before build — resolves the over-engineering and the cursor/@mention-drop bug in one cut. Medium.*

**F. INDEX v1 = static map + `brain status` pointer; Dataview → Phase 2.** Four lenses converge: the Dataview dashboard duplicates `brain status` in a second technology, needs Obsidian+plugin (not portable), and gets polluted by the `type: presence` template (phantom agent). Cut it. `brain status` becomes the single live-state source for human and agent. *Verdict: ADOPT before build — pure subtraction, removes a plugin dependency, de-risks Phase 2.*

**G. Nav skill lives machine-global; canonical source committed as a template.** Install to `~/.claude/skills/navigation-standards/` (like Steve's `/wrap` already is) with the source committed at `.brain/templates/navigation-standards.SKILL.md`. This dodges *both* verified problems: the repo's `.gitignore` line 59 silently ignores `.claude/skills/navigation-standards/`, **and** a committed skill wouldn't reach sibling feature branches until the init commit merges onto each. M8's "amend the nav skill" then edits the global file + commits the template copy; `/wrap` Phase-4 pathspec must include it. *Verdict: ADOPT before build — the governance loop is built on a file git won't track. High.*

**H. `brain init` settings + global hook dispatcher.** Init writes/merges the **committed** `.claude/settings.json` (none exists today) via `jq` append-dedupe into existing `hooks.*` arrays — never clobber, never touch `settings.local.json`. The **global** hook is a tiny dispatcher: `root="$(dirname "$(git rev-parse --git-common-dir 2>/dev/null)")"; [ -x "$root/.brain/bin/brain" ] && … || exit 0` — no-ops in every repo without a brain. *Verdict: ADOPT before build — as written, init is destructive in any non-empty repo and the global hook isn't project-agnostic. High.*

**I. Portable, lock-guarded reconcile — unpin durability from `/wrap` on parked `main`.** Extract `brain commit`/`brain reconcile` (path-scoped, `flock`-singleton, ff-only) that `/wrap` *calls* in one line. This fixes four findings at once: (1) `/wrap` is Steve's user-global skill (not portable); (2) the committer is currently hostage to the langgraph-on-`main` feature being awake; (3) two processes in the main worktree race on `index.lock`; (4) reconciliation runs only at `/wrap`. Move the *cheap* reconciliation (auto-resolve, abandoned-blocker downgrade) onto **SessionStart** so it's robust to the never-wraps agent. *Verdict: ADOPT before build — net-neutral complexity, large robustness gain. High.*

**J. Blocker-resolution by git fact + abandoned-blocker downgrade.** Detect "blocker shipped" via `git branch --merged origin/main` (catches UI-merges and crashed agents) OR presence `status: done`. Fire `waiting-on` auto-resolve on the **blocker's** reconcile (the session that knows it's done), not the blocked side. For a *stale/abandoned* blocker, downgrade `open→watch` + journal "blocker appears abandoned — proceeding" so a blocked feature is never wedged on a note nobody tends. *Verdict: ADOPT before build — the single most likely real-world wedge + the most likely silent wrong-unblock. High.*

**K. `ask` gate is forbidden when no human is attached.** The gate must detect non-interactive context (no TTY / autonomous) and force-downgrade `ask → allow + loud journal`. As written, an `ask` pause in the headless/cron mode the system targets is a literal M10 violation (run frozen until Steve returns). *Verdict: ADOPT before build — a no-TTY check; closes a real M10 hole. Medium-but-load-bearing.*

**L. Pre-authorize the resolved main `.brain/` path; state one-session-per-feature.** Once (A) points sibling agents at main's `.brain/`, they Write/Edit *outside their worktree* → the harness sandbox prompts on every coordination write. Init must allowlist that path in `settings.json`. Also state the invariant: one active session per feature, and **no sibling worktree ever stages/commits its local `.brain/`** (else the PR 3-way merge reverts evolving brain state). *Verdict: ADOPT before build — operational must once A lands; two sentences + one allowlist entry.*

## 3. Adopt — during build (implementation notes)

- **`touches` machine-derived, not hand-tended.** Reconcile/phase-boundary refreshes the agent's own `touches` from `git diff --name-only origin/main`; the PreToolUse gate computes *its own* side live from git and matches against others' presence `touches`. Cuts M9 upkeep and a stale-collision class. *(ADOPT — simplicity + consistency + superset converge.)*
- **`blocked_on` derived, not stored.** The blocked agent computes "am I blocked?" by reading referenced connections' live `status` at boot → a `resolved`/`watch` edge auto-clears with no cross-note write (removes the two-writer problem). *(ADOPT.)*
- **Demote churny live fields — except `current_worktree`.** `current_branch`/`current_ticket` → best-effort display, not a maintained contract. **Keep `current_worktree`** (it's the whoami tiebreaker from B) and mark it machine-local/ephemeral. *(ADOPT — partial; rejects simplicity's full demotion because worktree is load-bearing.)*
- **Deterministic connection slugs + `noclobber` create.** Slug = sorted feature-pair + kind → concurrent creation collapses to one file or a clean fail, never a silent dup. *(ADOPT — cheapest high-value concurrency fix.)*
- **`BRAIN_FEATURE` cached at SessionStart.** Reused by hooks (no re-shelling `whoami`) and doubles as the main-resident identity override from B. *(ADOPT.)*
- **Unify the pre-PR announce moment with the pre-PR hard gate** — one hook emits the "about to PR" line and runs the collision check. *(ADOPT — reuses machinery, drops a manual step.)*
- **Liveness gates on `status` first.** `active`/`blocked` always count; `idle`/`done` skip; `updated`-age is a secondary hint widened to ~5 days (Fri→Mon parks shouldn't age out), as one named constant. *(ADOPT — removes a false-negative class.)*
- **Boot surfacing filtered to `status: open|watch`.** Resolved is history you grep on demand, never injected — prevents months-long context bloat. *(ADOPT.)*
- **`kind` documented as a label that drives no logic** — guard against future branching on it. Keep all four (the seeds exercise each). *(ADOPT.)*
- **Structural proposals → a persistent `shared-rule` `watch` connection** + a journal event line pointing at it (not the ephemeral journal alone), and a `type:` allowlist check in `reconcile` flags stealth-structural notes. *(ADOPT — reuses connection machinery; resolves governance + superset.)*
- **Minimal secret scan in `brain commit`** — scan staged `.brain/` against the existing `data/canonical/tool_egress_denylist.yml` + `.env`-shaped regex; abort loudly on a hit. Research notes lift content *into* committed git, and this repo's own culture gates egress. *(ADOPT — lightweight, reuses an existing denylist.)*
- **Hook-error breadcrumb.** Fail-open hooks drop one line to `.brain/.hook-errors.log`; `brain status` greps it. Converts a silently-dead coordination layer into a detectable condition. *(ADOPT — keep it to the breadcrumb, no auto-heal.)*
- **`brain init` idempotent** — resolve canonical root first; if `<root>/.brain/` exists, no-op with notice (`--force` to re-template). Safe to run from any worktree. *(ADOPT — 3-line guard.)*
- **Drop the empty `.brain/.gitignore`** (ignores nothing in v1); reserve a `.indexignore` covering `templates/` + `bin/` for the Phase-2 superset, and folder-scope any future Dataview query. *(ADOPT — trivial, de-risks Phase 2.)*
- **Fold validation into `brain reconcile`** (not a new `brain doctor` command): flag colliding `owns_branches`, dangling `[[links]]`, stale-but-`active` features. *(ADOPT — enforcement point for B's collision guard.)*

## 4. Considered but rejected (and deferred)

**Rejected:**
- **Cut `brain new-feature` (simplicity).** REJECT — it's the deterministic seeding point for B's anchored wildcard and the action C's agent self-onboarding triggers. Hand-rolled presence notes are exactly the inconsistency that breaks identity.
- **Remove the `owns_branches` multi-pattern/append safety-net (simplicity).** REJECT — M6 is *violated by 3 of 4 live features today*; the multi-pattern path is required, not dead guidance.
- **Collapse `clash`→`shared-file` (simplicity, optional).** REJECT — keep the taxonomy; all four kinds are exercised by the seeds and the intent distinction aids the human glance (it's label-only cost).
- **Dedicated `last_journal_read` field (governance).** REJECT for v1 — only needed if the PostToolUse tier returns (deferred in E); infrequent-boot re-surfacing is acceptable. Revisit *with* tier-2.
- **Full structural-change authority/ratification model (governance).** REJECT the heavyweight version — the nav skill + a persistent proposal connection + the `type:` allowlist check suffice for a personal tool.
- **Separate `brain doctor` command (superset).** REJECT as a new command — fold the checks into `brain reconcile`.

**Deferred (CONSIDER at the right time, not now):**
- **`.brain/config.yml` repo-config seam (portability).** DEFER — this repo is the worked example; add the tracker-regex/branch-rule/plans-glob config when actually dropping into repo #2. Diffused prose is fine for a single worked target.
- **Wiki-link-valued relationship frontmatter + basename-uniqueness rule (superset).** DEFER to Phase-2 prep — partly mooted by deriving `blocked_on`; low priority until the graph layer lands.
- **Phase-2 index-DB `.gitignore` rule (portability).** DEFER — cheap to add when Phase 2 introduces the DB.
- **Atomic write-temp-then-`mv` for presence + `O_EXCL` journal header (superset/governance).** DEFER — presence is single-writer-per-feature (L) and journal append is already PIPE_BUF-safe; add only if a real clobber is observed.
- **`watch`-connection GC sweep (governance).** DEFER — the boot `open|watch` filter covers the bloat; auto-archiving stale `watch` zones is lower priority.

## 5. Top 3 residual risks (after adoption)

1. **Identity stays the fragile core.** Even with anchored matching + worktree tiebreak + `BRAIN_FEATURE`, the `main`-resident feature and any future work-on-`main` blur the branch-identity model; a mis-resolve is a *silent* wrong-note write (worst kind). The `refuse-ambiguous` path and the main-resident override need real-world watching, not just a unit test.
2. **The cross-worktree write model is operationally unproven.** "One live shared folder" depends on every agent writing into *main's* `.brain/` from a sibling worktree (permission allowlist holding) **and** no sibling ever staging its own checked-out `.brain/` copy. Tracked-`.brain` + per-worktree checkout is inherently in tension with "shared live folder"; a single mis-scoped commit or a denied write silently breaks sharing. This is the highest-leverage thing to smoke-test first.
3. **Self-policing has no external watchdog.** The system only reconciles when an agent boots or wraps. A fleet that crashes / never wraps, or a hook that dies silently after a nav-skill self-edit, degrades coordination toward zero with only the `.hook-errors.log` breadcrumb as signal. M9's "the system polices itself" is exactly as strong as boot/wrap frequency — which is adversarially low for parked features.
